API Reference

Every REST endpoint, grouped by service. Paths shown are the external gateway paths the SPA calls; the gateway strips the leading prefix before forwarding. Expand any endpoint for request/response detail and side effects.

Conventions

All paths except /auth/** and /user/create-user require Authorization: Bearer <jwt>. The therapistId / userId are read from JWT claims server-side via SecurityUtils — clients never send them in the body (a field marked “from JWT” below is populated by the server and ignored if sent). The gateway routes by prefix: /therapist/* → therapist-service, /appointment/* → appointment-service, /client/* → client-service, /user/* → user-service, /analytics/* → analytics-service.

Authentication gateway · /auth

POST/auth/loginAuthenticate & issue tokens

Auth: public · rate-limited to 10 req/min (HTTP 429 via loginRateLimitFallback).

Request

AuthRequest{ "username": "jdoe", "password": "••••••••" }

Responses

  • 200 — { "token": "<jwt>" } plus Set-Cookie: refresh_token (httpOnly, Secure, SameSite=Strict, path /auth/refresh, 7-day).
  • 401 — { "failureReason": "INVALID_CREDENTIALS | ACCOUNT_LOCKED | ACCOUNT_DISABLED | SERVICE_UNAVAILABLE" }
  • 429 — { "error": "Too many login attempts…" }

Flow & side effects: validates via UserServiceProxy.validateUser (circuit-broken); publishes auth-login-success/auth-login-failure to Kafka; resolves therapistId from TherapistService; signs a 15-min RS256 JWT; persists a REFRESH_TOKENS row (deleting any prior tokens for the user).

POST/auth/refreshRotate refresh token, issue new access token

Auth: refresh_token cookie · Request: none (cookie only).

Responses: 200 { "token": "<jwt>" } + new rotated cookie · 401 { "error": "No refresh token" | "Invalid or expired refresh token" }.

Side effects: deletes the presented token and issues a fresh one (rotation).

Known gap: roles aren’t stored on the refresh-token row, so tokens minted here carry an empty authorities claim. See Technical Debt.

POST/auth/logoutRevoke refresh token

Auth: refresh cookie (optional) · Response: 200 always, with a cleared cookie (maxAge=0).

Side effects: deletes the user’s refresh tokens (revocation) if the cookie is present.

POST/auth/forgot-passwordRequest a reset email

Auth: public · Request: { "email": "user@example.com" }

Response: 200 { "message": "If the email exists, reset instructions will be sent." } — generic by design (no account enumeration). Proxies to UserService.

POST/auth/reset-passwordComplete a password reset

Auth: public (token-scoped) · Request: { "token": "…", "newPassword": "…" }

Response: 200 { "message": "Password reset successfully." }. Token must be unused and within its 30-minute window. Proxies to UserService.

User user-service · /user

POST/user/create-userRegister a user

Auth: public · Request

UserDto{ "username": "jdoe", "email": "jdoe@x.com",
  "password": "••••••••", "userRole": "THERAPIST" }

Response: 200 (empty body). BCrypt-hashes the password; generates a USR… id.

POST/user/validate-userCredential check (gateway-internal)

Auth: permitAll (intended for gateway use only) · Request: { "username", "password" }.

Response

AuthResponse{ "authenticated": true, "userId": "USR…", "username": "jdoe",
  "roles": ["THERAPIST"], "accountStatus": { "enabled": true, "locked": false },
  "failureReason": null }

On failure: authenticated=false and failureReason = INVALID_CREDENTIALS | ACCOUNT_DISABLED | ACCOUNT_LOCKED.

POST/user/forgot-passwordIssue reset token + email

Auth: public · Request: { "email": "…" } · Response: 200 generic message.

Side effects: for a known email, stores a 30-min resetPasswordToken and sends a link via the Gmail API (EmailService); silently no-ops for unknown emails.

POST/user/reset-passwordSet new password from token

Auth: public · Request: { "token": "…", "newPassword": "…" } · Response: 200. Rejects invalid/expired tokens (IllegalArgumentException → handled by GlobalExceptionHandler). Clears the token on success.

PUT/user/update-accountUpdate email/username

Auth: JWT (username resolved from token, not body) · Request

UpdateAccountRequest{ "username": "newname", "email": "new@x.com", "currentPassword": "••••" }

Response: 200 updated UserDto. Requires the correct currentPassword; blank fields are ignored.

Services & delivery modes

GET/therapist/therapist-servicesList services (paged optional)

Auth: JWT · Query: optional ?page&size → returns a PageResponseDto instead of a list · Response item: TherapistServicesDto (serviceId, serviceType, duration, price, isActive).

Auth: JWT · Path: therapistId · Response: List<TherapistServicesDto>.

Auth: JWT · Request

TherapistServicesDto{ "serviceType": "INDIVIDUAL_THERAPY", "duration": 60,
  "price": 120.00, "isActive": true }   // therapistId from JWT

Response: 200. serviceType ∈ {INDIVIDUAL_THERAPY, COUPLES_THERAPY}. duration drives slot length during generation.

PUT/therapist/update-service/{serviceId}Update a service

Auth: JWT · Path: serviceId · Request: TherapistServicesDto · Response: updated dto.

DELETE/therapist/delete-service/{serviceId}Delete a service

Auth: JWT · Response: 200 "Therapist service deleted successfully."

GET/therapist/delivery-modesList delivery modes

Auth: JWT · Query: optional ?serviceId to filter · Response: List<TherapyDeliveryModeDto>.

POST/therapist/delivery-modesCreate a delivery mode

Auth: JWT · Request

TherapyDeliveryModeDto{ "serviceId": "…", "modeType": "ONLINE",   // ONLINE | OFFLINE
  "displayName": "Video call", "address": null, "price": 120.00, "isActive": true }

Response: 200 created dto. Pricing/mode here is what AppointmentService validates against at booking. Emits a delivery-mode event so AppointmentService updates its projection.

PUT/therapist/delivery-modes/{modeId}Update a delivery mode

Auth: JWT · Request: TherapyDeliveryModeDto · Response: updated dto.

DELETE/therapist/delivery-modes/{modeId}Delete a delivery mode

Availability

GET/therapist/availability-rulesList weekly rules

Auth: JWT · Query: optional ?page&size · Response item: TherapistAvailabilityRulesDto (ruleId, dayOfWeek 1–7, startTime, endTime, isActive).

POST/therapist/create-availability-rulesCreate weekly rules (bulk)

Auth: JWT · Request: a list of rules

List<TherapistAvailabilityRulesDto>[ { "dayOfWeek": 1, "startTime": "09:00", "endTime": "17:00", "isActive": true } ]

Response: 200. therapistId is stamped on each from the JWT.

PUT/therapist/availability-rules/{ruleId}Update a rule

Auth: JWT · Request: TherapistAvailabilityRulesDto · Response: updated dto.

DELETE/therapist/availability-rules/{ruleId}Delete a rule

Auth: JWT · Response: 200 "Availability rule deleted successfully."

POST/therapist/create-availability-overridesSingle date-specific override

Auth: JWT · Request

TherapistAvailabilityOverridesDto{ "startTime": "2026-07-10T09:00:00", "endTime": "2026-07-10T12:00:00",
  "isAvailable": false, "reason": "Conference", "syncToGoogleCalendar": true }

Response: 200. Same-day windows only. An unavailable override blocks bookings; an available one adds slots. May emit a CalendarBlockEvent when syncToGoogleCalendar is set.

POST/therapist/bulk-availability-overridesBulk overrides over a date range

Auth: JWT · Request

BulkAvailabilityOverridesRequest{ "startDate": "2026-07-10", "endDate": "2026-07-14",
  "isAvailable": false, "reason": "Leave", "syncToGoogleCalendar": false }

Response: 200 "Bulk availability overrides created successfully."

GET/therapist/availability-overridesList overrides

Auth: JWT · Response: List<TherapistAvailabilityOverridesDto>.

DELETE/therapist/availability-overrides/{overrideId}Delete an override

Auth: JWT · Response: 200 "Availability override deleted successfully."

POST/therapist/generate-slotsExpand rules+overrides into slots

Auth: JWT · Query: startDate, endDate (ISO date) · Response: 200 "Successfully generated slots" (or 400 if endDate < startDate / overlaps active appointments).

Side effects: deletes existing slots in range, regenerates from active rules × services (30-min step), applies available overrides, and emits AvailabilitySlotsGenerated → AppointmentService projection.

DELETE/therapist/delete-slotsDelete slots in a range

Auth: JWT · Query: startDate, endDate · Response: 200/400. Refuses if active appointments overlap; emits AvailabilitySlotsDeleted.

Clients, notes, dashboard & earnings

Auth: JWT · Request: ClientDto (firstName, lastName, dob, phoneNumber, emergencyPhoneNumber, email, pronouns, gender, dsf).

GET/therapist/clientsClient roster

Auth: JWT · Path: clientId · Request

SessionNotesDto{ "appointmentId": "…", "clientId": "CLT…", "sessionNotes": "…" }

Response: 200. Content is AES-GCM encrypted at rest via SessionNotesEncryptor. One note per appointment (unique).

PUT/therapist/{clientId}/update-notesUpdate session notes

Auth: JWT · Request: SessionNotesDto · Response: 200.

GET/therapist/{clientId}/session-detailsClient appointment history

Auth: JWT · Response: List<SessionDetailsDto> built from the appointment projection.

GET PUT/therapist/{clientId}/noteGet / upsert per-client note

Auth: JWT · GET Response: ClientNotesDto (content, createdAt, updatedAt) · PUT Request: ClientNotesDto → upserted note.

  "dayEarnings", "weekEarnings", "monthEarnings", "lifetimeEarnings" }

EarningsSummaryDto{ week/month/lifetime Earnings,

GET/therapist/earnings/sessionsEarnings session list

Auth: JWT · Query: fromDate, toDate (required), serviceId?, modeId? · Response: List<EarningsSessionDto>. Returns 400 if toDate < fromDate.

GET/therapist/earnings/exportCSV export

Auth: internal (gateway over Eureka: http://therapist-service/internal/therapist/user/{userId}) · Response: the therapistId string. Called during login to embed the therapistId claim in the JWT.

Appointment appointment-service · /appointment

POST/appointment/create-appointmentBook a slot

Auth: JWT (therapistId from token) · Request

BookAppointmentRequest{ "slotId": "SLT…", "clientId": "CLT…", "modeId": "…", "clientName": "Jane" }

Response: 200 appointmentId. Errors: SlotNotAvailableException (unknown/blocked slot or invalid mode), SlotAlreadyBookedException (lost the race).

Flow: validates slot + unavailable overrides + active delivery mode (→ price), atomically markSlotAsBooked (0 rows ⇒ already booked), persists the appointment, emits AppointmentCreated via the outbox.

PATCH/appointment/update-appointmentStatus transition

Auth: JWT · Request

UpdateAppointmentStatusRequest{ "appointmentId": "…", "status": "CONFIRMED", "reason": null }

Allowed targets: CONFIRMED | COMPLETED | CANCELLED | ABANDONED. Terminal states are immutable. CANCELLED releases the slot. Emits the matching event (AppointmentConfirmed, …). Errors: InvalidAppointmentStatusTransitionException, AppointmentNotFoundException.

PATCH/appointment/reschedule-appointmentMove to a new slot

Auth: JWT · Request

RescheduleAppointmentRequest{ "appointmentId": "…", "newSlotId": "SLT…", "modeId": "…"?, "reason": "…"? }

Response: 200 "Appointment rescheduled". Books the new slot, releases the old, re-prices by mode, sets status RESCHEDULED, emits AppointmentRescheduled (with old/new slot + times). Rejects reschedules of terminal appointments or to the same slot.

GET/appointment/get-appointmentsToday’s active appointments

Auth: JWT · Response: List<TherapistAppointments> — today’s CONFIRMED/RESCHEDULED, ordered by start time.

GET/appointment/get-availabilityEffective slots + appointments

Auth: JWT · Response: List<AvailabilityResponseDto> — effective slots joined with any booked appointment.

GET/appointment/editor-viewCalendar editor view

Auth: JWT · Query: fromDate, toDate · Response: AppointmentScheduleViewDto = { slots, appointments, overrides } for the range. 400 if toDate < fromDate.

GET/appointment/searchFiltered appointment search

Auth: JWT · Query: clientName?, status? (repeatable AppointmentStatus), fromDate, toDate · Response: List<AppointmentScheduleAppointmentDto>. Empty status ⇒ all statuses.

Error handling

UserService, ClientService, TherapistService and AppointmentService each register a GlobalExceptionHandler (@RestControllerAdvice) mapping domain exceptions to a consistent ApiError body. Appointment-specific exceptions: SlotNotAvailableException, SlotAlreadyBookedException, AppointmentNotFoundException, InvalidAppointmentStatusTransitionException, AvailabilityProjectionConflictException.
