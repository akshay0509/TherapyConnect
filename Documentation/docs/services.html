DiscoveryService :8761

A bare Netflix Eureka server (@EnableEurekaServer). It registers nothing itself (register-with-eureka=false, fetch-registry=false) and holds no business logic — it is the registry every other service joins so the gateway can route via lb://SERVICE-NAME. Start it first.

GatewayService :8091

The single entry point for the SPA. Combines Spring Cloud Gateway (MVC) routing with custom authentication endpoints.

Responsibilities

  • Routing — path-prefix routes to backend services, stripping the prefix. The therapist route also injects the X-Therapist-ID header from the path.
  • Authentication — AuthController (/auth/**) handles login, refresh, logout, forgot/reset password; AdminAuthController handles /auth/admin-login (constant-time env-credential check, 8-hour ADMIN token).
  • Token issuance — JwtUtil signs RS256 access tokens (15-min TTL); RefreshTokensService manages rotating 7-day refresh tokens (roles preserved) in a SameSite=None httpOnly cookie.
  • Authorization — JwtAuthenticationConverter maps the authorities claim; /admin/** requires ADMIN; /*/internal/** is denied at the edge.
  • Admin dashboard — AdminServicesController (/admin/services) reports per-service health from Eureka + actuator.
  • Resilience — a custom per-IP LoginRateLimiter throttles login (10/60s) and admin login (5/60s) by client IP; Resilience4j circuit breakers wrap the User and Therapist proxy calls.
  • Login eventing — publishes auth-login-success/auth-login-failure directly to Kafka (no outbox). Declares all Kafka topics via TopicsConfig.

/therapist/{tid}/**, /therapist/**, /client/**, /user/**, /appointment/**, /analytics/**

UserService :5003

System of record for accounts and credentials. The gateway delegates credential validation here during login but issues the token itself.

Responsibilities

  • Create users (BCrypt password hashing via PasswordConfig).
  • Validate credentials — returns a structured AuthResponse with success/failure reason (invalid credentials, disabled, locked).
  • Password reset — issue a 30-minute reset token and email a link via Gmail API (EmailService).
  • Update account (email/username) — identity taken from the JWT, not the body.
  • Forgot username — emails the username for a given address (generic response, no enumeration).
  • Consume login events to maintain audit trail & lockout: 5 failed attempts locks the account; success resets the counter.
  • Admin user management — AdminUserController: list users, enable/lock, force-reset password, recent login audit.

TherapistService :5002

The largest and most central domain service. Owns the therapist profile and everything the therapist configures, plus read models for clients and appointments that power the dashboard.

Responsibilities

  • Therapist profile CRUD; resolve therapistId from userId (the gateway calls this during login).
  • Services (e.g. duration, price) and delivery modes (online/offline, address, price).
  • Payment settings & timezone — PUT /payment-settings toggles Razorpay collection; the profile carries a timezone. Both changes emit a TherapistEvent that AppointmentService projects.
  • Availability rules (weekly recurring) and overrides (date-specific add/remove); expands them into concrete slots via AvailabilitySlotService.
  • Dashboard stats, earnings summary/sessions, and CSV export of earnings.
  • Client roster (projection) and creating clients (delegates to ClientService over REST).
  • Encrypted session notes (EncryptionService AES-GCM + JPA @Convert) and per-client notes.
  • Propagates the caller's JWT on outbound REST calls (JwtPropagationInterceptor) and adds trace IDs to logs (MdcFilter/TraceIdInterceptor).

AppointmentService :5005

The booking engine and authority on appointment state. It books against a local projection of availability slots (kept by consuming availability events) and emits appointment events that everyone else reacts to.

Responsibilities

  • Book an appointment — validate slot availability & delivery mode, atomically mark the slot booked (optimistic markSlotAsBooked returning affected-rows to detect races), persist the appointment, emit AppointmentCreated.
  • Status transitions — confirm / complete / cancel / abandon with a guarded state machine (terminal states are immutable; cancel releases the slot).
  • Reschedule — move to a new slot, release the old one, re-price by mode.
  • Read APIs — today's appointments, availability-with-appointments, an editor calendar view, and search.
  • Maintains projections of availability slots and delivery modes; refuses destructive availability syncs that would clobber booked slots.
  • Payments — owns the Razorpay payment-link lifecycle (PaymentService, RazorpayLinkClient): creates a link on booking, cancels on cancel, reissues on reschedule, and an HMAC-verified webhook auto-confirms paid appointments. Fails closed when Razorpay keys or the therapist's paymentEnabled flag are off.
  • Admin — /admin/health and /admin/outbox/replay (operational re-publish of outbox events).

🛡️ Admin console (cross-service)

A single admin surface spans several services, all gated behind ADMIN authority (login at /auth/admin-login): the Gateway exposes /admin/services (Eureka + actuator health dashboard); UserService exposes user management (/admin/users, status, force reset, /admin/audit); Appointment & Analytics expose /admin/health, and Appointment adds /admin/outbox/replay. The React app renders these in AdminPage. See the API reference.

Events library

Not a runtime service — a shared JAR (com.org:events) holding the event POJOs and enums every service serializes/deserializes. It is the cross-service contract; see Messaging & Events for the schemas.

Frontend

auth-app/ is a React 18 + Vite SPA (React Router, Axios, Recharts for charts, Lucide icons). It is the only client and talks exclusively to the gateway. It is not a backend module and is excluded from the parent POM build.
