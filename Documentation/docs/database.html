Data Model

JPA entities, ownership, relationships and projections. Schema is generated by Hibernate (ddl-auto=update) from these entities against PostgreSQL.

⚠️ No migrations

There are no Flyway/Liquibase migrations. Tables are created/altered by Hibernate at startup from the entity classes below. The entity is therefore the source of truth for the schema. Relationships between services' tables are logical (foreign keys by ID string), not enforced DB constraints across services.

Database per service

One PostgreSQL 16 instance hosts seven separate databases — gateway, user, therapist, client, appointment, notification, analytics — created on first boot by postgres-init/init-databases.sql. Each service connects only to its own database (jdbc:postgresql://postgres:5432/<db>). Isolation is now physical at the database level, though the services still share one credential pair.

ID conventions

Most entities use a string primary key with a service-specific prefix, generated in a JPA @PrePersist hook from a truncated UUID. This makes IDs self-describing across the event bus.

USER_LOGIN_AUDIT

Append-only audit of every login attempt (success and failure), with indexes on userId, username and loginAt. Written by the login consumers.

GatewayService — REFRESH_TOKENS

One active refresh token per user (creating a new one deletes prior tokens for that username). Stores userId, therapistId and roles so /auth/refresh can reissue a fully-scoped JWT without extra service calls. Columns: id PK, token (unique), username, expiryDate (LocalDate, +7 days), revoked, userId, therapistId, roles. (The roles column was added to fix refreshed tokens losing their authorities.)

🔒 Field-level encryption

SESSION_NOTES.noteContent uses a JPA AttributeConverter (SessionNotesEncryptor) backed by EncryptionService — AES/GCM/NoPadding, 12-byte random IV prepended, 128-bit tag, key from app.encryption.secret (Base64). Notes are unreadable at rest without the key.

The OUTBOX_EVENT table

Identical shape in ClientService, TherapistService and AppointmentService. The payload is stored as PostgreSQL jsonb via Hypersistence Utils (JsonBinaryType).
