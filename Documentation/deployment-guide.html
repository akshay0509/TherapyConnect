TherapyConnect — Docker Deployment Guide

Oracle Cloud VM.Standard.A1.Flex (ARM64) · Ubuntu 24.04 · Java 25 · Docker Compose

Overview

The stack runs entirely on a single Oracle Free Tier instance via Docker Compose. Nginx on the host forwards port 80 → Gateway container on 8091. All service containers share a private therapy-net bridge network.

Phase 1 — Local Windows Machine

Apply all proposed changes and push to GitHub before touching the server.

1
Apply proposed code changes to the repo

Copy all files from D:\Claude\ProposedChanges\ into D:\GIT\repository\, preserving the folder structure. This applies the technical debt fixes (roles-on-refresh, DLQ, logging, etc.).

Files that already exist get overwritten. New files (Dockerfiles, docker-compose.yml, KafkaConsumerConfig.java) get created.

2
Add a root .gitignore entry for the keys folder

Open D:\GIT\repository\.gitignore (create it if it doesn't exist) and add:

keys/

This prevents the PEM keys from ever being committed to GitHub.

3
Push everything to GitHub
cd D:\GIT\repository
git add .
git commit -m "Add Docker files and apply proposed changes"
git push

Phase 2 — Oracle Server (SSH session)

Run all remaining commands as the ubuntu user inside /home/ubuntu/TherapyConnect.

4
Pull the latest code
cd /home/ubuntu/TherapyConnect
git pull

This brings in the Dockerfiles, docker-compose.yml, and the proposed code changes.

5
Rebuild all JARs

Required because the ProposedChanges modified Java source files and pom.xml files.

export JAVA_HOME=/opt/jdk-25
export PATH=$JAVA_HOME/bin:$PATH
mvn clean package -DskipTests

This takes 3–5 minutes. All 8 service JARs will be in their respective target/ directories.

6
Build the JDK 25 base Docker image

This is a one-time step. It downloads Microsoft JDK 25 (~200 MB) and creates a local image called therapy-jdk25. All service images will be based on this.

docker build -t therapy-jdk25 -f Dockerfile.base .

Takes 3–5 minutes (downloads JDK). Subsequent rebuilds use the Docker layer cache and are instant.

7
Copy the PEM keys to the server

Open a new terminal on your Windows machine (keep the SSH session open). Run these SCP commands, replacing <server-ip> and <path-to-ssh-key>:

# On Windows PowerShell
$SERVER = "<your-oracle-public-ip>"
$SSH_KEY = "C:\Users\ASUS\path-to-your-oracle-ssh-key.key"

# Create the keys directory on the server first
ssh -i $SSH_KEY ubuntu@$SERVER "mkdir -p /home/ubuntu/TherapyConnect/keys"

# Copy both PEM files
scp -i $SSH_KEY "D:\GIT\repository\GatewayService\src\main\resources\keys\gateway-private.pem" ubuntu@${SERVER}:/home/ubuntu/TherapyConnect/keys/
scp -i $SSH_KEY "D:\GIT\repository\GatewayService\src\main\resources\keys\gateway-public.pem"  ubuntu@${SERVER}:/home/ubuntu/TherapyConnect/keys/
Important: Never commit PEM keys to git. They are already gitignored at the repo root after Step 2.

Back in the SSH session, verify the keys arrived:

ls /home/ubuntu/TherapyConnect/keys/
# Expected: gateway-private.pem  gateway-public.pem
8
Create the .env file with your secrets
cd /home/ubuntu/TherapyConnect
cp .env.example .env
nano .env

Fill in all blank values:

VariableWhere to get it
DB_PASSWORDChoose any strong password — Postgres will be initialised with it
GOOGLE_CLIENT_IDGoogle Cloud Console → Credentials → OAuth 2.0 Client ID
GOOGLE_CLIENT_SECRETSame credential
GOOGLE_REFRESH_TOKENOAuth Playground or your existing token
THERAPIST_EMAILThe therapist's Google calendar email
GMAIL_FROMThe Gmail address used to send emails
APP_ENCRYPTION_SECRETGenerate: openssl rand -base64 32

DB_USERNAME defaults to postgres — keep it unless you have a reason to change it.
9
Build all service Docker images
cd /home/ubuntu/TherapyConnect
docker compose build

This copies each pre-built JAR into its image. Fast — about 30–60 seconds total since JDK 25 is already in the base layer cache.

10
Start everything
docker compose up -d

Docker Compose starts services in dependency order:

  1. Postgres and Kafka start simultaneously
  2. Discovery (Eureka) starts once infra is healthy
  3. All 7 business services start once Eureka is healthy

First startup takes 3–4 minutes. Eureka typically takes 60 seconds to be ready.

Verification

11
Check all containers are running
docker compose ps

All 10 containers should show Up status. If any show Restarting, check its logs:

docker compose logs --tail=50 gateway
docker compose logs --tail=50 user
12
Check Eureka dashboard

In a browser, open:

http://<your-oracle-public-ip>:8761
Port 8761 is exposed from the discovery container directly to the host. Make sure it's allowed in the Oracle Security List — add it the same way you added port 80 and 8091 earlier.

You should see all 7 services registered: GATEWAY-SERVICE, USER-SERVICE, THERAPIST-SERVICE, CLIENT-SERVICE, APPOINTMENT-SERVICE, NOTIFICATION-SERVICE, ANALYTICS-SERVICE.

13
Test the API through Nginx
# Health check through gateway
curl http://<your-oracle-public-ip>/actuator/health

# Or just verify the gateway responds
curl -I http://<your-oracle-public-ip>/
Expected: A response (even 401 Unauthorized) means the full chain is working: Browser → Nginx → Gateway container → Eureka → Target service.

Common Issues

Service keeps restarting

docker compose logs --tail=100 <service-name>

Most likely causes: wrong .env value, missing key file, or Eureka not yet ready (the service will retry automatically — wait 2 minutes).

Kafka health check failing

Kafka takes 30–60 seconds to initialise. Dependent services will wait due to condition: service_healthy. If it stays unhealthy after 2 minutes:

docker compose logs kafka

Services not appearing in Eureka

Services register with Eureka 30 seconds after startup (default heartbeat). If they never appear:

docker compose logs user | grep -i eureka

Look for Registered instance or connection errors.

JWT key not found error

Verify the keys are in the right place and readable:

ls -la /home/ubuntu/TherapyConnect/keys/
# Both files must exist with read permission

Rebuilding after a code change

# 1. Build new JARs
mvn clean package -DskipTests

# 2. Rebuild and restart only the changed service (e.g., gateway)
docker compose build gateway
docker compose up -d gateway
